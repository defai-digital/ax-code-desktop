# Environment Variables

AX Code App reads these environment variables at startup. For startup services, `openchamber startup enable` snapshots the current environment by default, so rerun it after changing variables you want the service to use.

## AX Code App server

### `OPENCHAMBER_HOST`

Bind address for the AX Code App web server. Use `0.0.0.0` to allow access from other machines.

### `OPENCHAMBER_UI_PASSWORD`

Password for the browser UI. Use this when binding outside localhost, using tunnels, or running behind a reverse proxy.

### `OPENCHAMBER_DATA_DIR`

Overrides the AX Code App data directory. The default is `~/.config/openchamber`.

### `OPENCHAMBER_COMPRESS_API`

Controls API response compression. Use `true` or `1` to force it on, `false` or `0` to force it off.

### `OPENCHAMBER_SKIP_API_COMPRESSION`

Disables API response compression when set to `true` or `1`. This takes precedence over `OPENCHAMBER_COMPRESS_API`.

### `OPENCHAMBER_VERBOSE_REQUEST_LOGS`

Enables verbose HTTP request logs when set to `true` or `1`.

### `OPENCHAMBER_UPDATE_API_URL`

Overrides the update-check API endpoint. Most users should leave this unset.

### `OPENCHAMBER_PACKAGE_MANAGER`

Forces the package manager used by update operations when auto-detection is wrong.

## AX Code server

### `AX_CODE_HOST`

Connects AX Code App to an existing AX Code server. The value must be an `http` or `https` origin with an explicit port and no path, query, or hash. `AX_CODE_HOST` takes precedence over `AX_CODE_PORT`.

### `AX_CODE_PORT`

Sets the AX Code server port. With managed AX Code this requests the managed port; with `AX_CODE_SKIP_START=true` it connects to an external server on that port.

### `AX_CODE_SKIP_START`

Prevents AX Code App from starting its own AX Code server when set to `true`.

### `OPENCHAMBER_AX_CODE_HOSTNAME`

Bind hostname for the AX Code server managed by AX Code App. The default is `127.0.0.1`.

### `AX_CODE_BINARY`

Path to the `ax-code` executable AX Code App should run.

### `AX_CODE_CONFIG`

Path to a specific AX Code config file.

### `AX_CODE_CONFIG_DIR`

Path to a specific AX Code config directory for agents, skills, snippets, and config discovery.

### `AX_CODE_DATA_DIR`

Custom data directory for the managed AX Code server.

### `AX_CODE_WSL_DISTRO`

Selects the WSL distro used for AX Code integration on Windows.

### `OPENCHAMBER_AX_CODE_WSL_DISTRO`

AX Code App-specific alias for selecting the WSL distro. `AX_CODE_WSL_DISTRO` takes precedence when both are set.

### `AX_CODE_JWT_SECRET`

Secret used to sign UI authentication tokens. Use a long random value for persistent service deployments.

## Terminal and Git

### `OPENCHAMBER_TERMINAL_SHELL`

Shell executable used for AX Code App terminal sessions.

### `OPENCHAMBER_GIT_BINARY`

Git executable used by AX Code App Git features.

### `GIT_BINARY`

Alternative Git executable override. Prefer `OPENCHAMBER_GIT_BINARY` for AX Code App-specific configuration.

### `OPENCHAMBER_GIT_READ_CACHE_TTL_MS`

Time-to-live, in milliseconds, for cached Git-backed file reads. Set `0` to disable this cache while debugging.

## Voice and tunnels

### `OPENAI_API_KEY`

API key used by AX Code App voice features that call OpenAI-compatible services.

### `OPENCHAMBER_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Allows remote OpenAI-compatible base URLs for voice features when set to `true` or `1`.

### `NGROK_AUTHTOKEN`

ngrok auth token used by AX Code App tunnel commands. You can also configure ngrok with `ngrok config add-authtoken <token>`.

## Runtime helpers

### `BUN_BINARY`

Bun executable AX Code App should use when spawning daemon processes.

### `BUN_INSTALL`

Bun installation root. AX Code App uses it to find `bin/bun` for daemon startup and updates.

### `VITE_AX_CODE_URL`

Build-time API base URL for the Vite-built web app. Most users should not set it for normal CLI or desktop usage.
