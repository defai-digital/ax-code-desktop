# AX Code Server

AX Code App se ejecuta sobre un servidor de AX Code. De forma predeterminada inicia uno por ti, así que no tienes que hacer nada. Solo necesitas esta página si quieres apuntar AX Code App a un servidor que ya ejecutas, o gestionar el que él inicia.

## Cómo encuentra AX Code App un servidor

Cuando AX Code App arranca, busca un servidor en este orden:

1. reutilizar un servidor que ya inició
2. conectarse a uno externo si se lo indicaste (ver más abajo)
3. detectar automáticamente un servidor en el puerto predeterminado (`4096`)
4. de lo contrario, iniciar y gestionar el suyo propio

Si no hay nada configurado, el paso 4 ocurre automáticamente y ya tienes todo en marcha.

## Conéctate a un servidor que ya ejecutas

Define esto antes de iniciar AX Code App:

```bash
AX_CODE_HOST=http://localhost:4096 AX_CODE_SKIP_START=true ax-code-openchamber
```

- `AX_CODE_HOST` — la dirección completa de tu servidor de AX Code, incluido el puerto (un valor como `http://localhost:4096`). No debe tener una ruta al final.
- `AX_CODE_SKIP_START=true` — le indica a AX Code App que no inicie su propio servidor.

Si solo necesitas cambiar el puerto, define `AX_CODE_PORT` en lugar de `AX_CODE_HOST`.

Si a `AX_CODE_HOST` le falta el puerto o tiene una ruta, AX Code App lo ignora y recurre a iniciar su propio servidor. Observa los registros de arranque por si aparece una advertencia `[config]` cuando una conexión que esperabas no ocurrió.

## Gestiona el servidor desde la CLI

```bash
openchamber status
openchamber logs
openchamber restart
openchamber stop
```

`openchamber` por sí solo inicia el servidor en segundo plano. Añade `--foreground` para mantenerlo asociado a tu terminal.

## Iniciar AX Code App al iniciar sesión

Usa `startup enable` para instalar un servicio nativo de usuario. AX Code App usa `launchd` en macOS, `systemd --user` en Linux y Task Scheduler en Windows.

```bash
openchamber startup enable
openchamber startup status
openchamber startup disable
```

Para proteger la UI, define la contraseña al habilitar el servicio:

```bash
OPENCHAMBER_UI_PASSWORD='secret' openchamber startup enable
```

`startup enable` guarda una captura del entorno actual en el servicio para que el arranque se parezca más a ejecutar `openchamber` desde la misma shell. Así conserva tokens de proveedores, `PATH`, configuración del agente SSH y otras variables CLI de auth/config. Usa `--no-env-snapshot` si quieres un entorno de servicio mínimo.

Puedes gestionar túneles de forma independiente para ese servicio en ejecución:

```bash
openchamber tunnel start --port 3000
openchamber tunnel stop --port 3000
```

Detener el túnel no reinicia el servicio ni la app.

## "AX Code is restarting"

Mientras el servidor se inicia o reinicia, AX Code App muestra un estado "AX Code is restarting" y pausa las solicitudes hasta que esté listo. Esto es normal justo después de iniciar o reiniciar. Si nunca desaparece, consulta [Conexión de AX Code](/es/troubleshooting/ax-code-connection/).
