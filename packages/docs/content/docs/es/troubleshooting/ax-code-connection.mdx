# Conexión de AX Code

Si AX Code App carga pero nunca pasa de "AX Code is restarting", o el chat no responde, el servidor con el que se comunica no es accesible. Repasa estos pasos.

## Se queda atascado en "AX Code is restarting"

- dale un momento justo después de iniciar: este estado es normal mientras el servidor arranca
- comprueba si el servidor está vivo con `openchamber status`
- reinícialo con `openchamber restart`
- ve los detalles de arranque con `openchamber logs`

## Te conectas a tu propio servidor

Si configuraste AX Code App para usar un servidor existente, revisa de nuevo la configuración de [AX Code Server](/es/ax-code-server/):

- `AX_CODE_HOST` debe incluir el puerto, sin ruta; por ejemplo `http://localhost:4096`
- define `AX_CODE_SKIP_START=true` para que AX Code App no inicie también el suyo
- si la dirección es inválida, AX Code App la ignora e inicia su propio servidor en su lugar; busca una advertencia `[config]` en los registros

## Sigue fallando

- confirma que Node.js sea la versión `20` o más reciente
- reinstala la última CLI
- abre `http://localhost:3000` directamente antes de probar cualquier túnel o enlace remoto
