# Variables de entorno

AX Code App lee estas variables al arrancar. Para servicios de inicio, `openchamber startup enable` guarda por defecto una copia del entorno actual, así que vuelve a ejecutarlo después de cambiar variables que quieras usar en el servicio.

## Servidor AX Code App

### `OPENCHAMBER_HOST`

Dirección donde escucha el servidor web de AX Code App. Usa `0.0.0.0` para permitir acceso desde otras máquinas.

### `OPENCHAMBER_UI_PASSWORD`

Contraseña para la interfaz del navegador. Úsala cuando no te limites a localhost, o cuando uses túneles o un proxy inverso.

### `OPENCHAMBER_DATA_DIR`

Cambia el directorio de datos de AX Code App. Por defecto es `~/.config/openchamber`.

### `OPENCHAMBER_COMPRESS_API`

Controla la compresión de respuestas de la API. `true` o `1` la activa; `false` o `0` la desactiva.

### `OPENCHAMBER_SKIP_API_COMPRESSION`

Desactiva la compresión de respuestas de la API si vale `true` o `1`. Tiene prioridad sobre `OPENCHAMBER_COMPRESS_API`.

### `OPENCHAMBER_VERBOSE_REQUEST_LOGS`

Activa logs HTTP detallados si vale `true` o `1`.

### `OPENCHAMBER_UPDATE_API_URL`

Cambia el endpoint de comprobación de actualizaciones. La mayoría de usuarios no debería tocarlo.

### `OPENCHAMBER_PACKAGE_MANAGER`

Fuerza el gestor de paquetes usado para actualizar cuando la detección automática se equivoca.

## Servidor AX Code

### `AX_CODE_HOST`

Conecta AX Code App a un servidor AX Code existente. Debe ser un origin `http` o `https` con puerto explícito y sin ruta, query ni hash. `AX_CODE_HOST` tiene prioridad sobre `AX_CODE_PORT`.

### `AX_CODE_PORT`

Define el puerto del servidor AX Code. Con AX Code gestionado pide ese puerto; con `AX_CODE_SKIP_START=true` conecta a un servidor externo en ese puerto.

### `AX_CODE_SKIP_START`

Evita que AX Code App arranque su propio servidor AX Code cuando vale `true`.

### `OPENCHAMBER_AX_CODE_HOSTNAME`

Hostname para el servidor AX Code gestionado por AX Code App. Por defecto es `127.0.0.1`.

### `AX_CODE_BINARY`

Ruta al ejecutable `ax-code` que debe usar AX Code App.

### `AX_CODE_CONFIG`

Ruta a un archivo de configuración concreto de AX Code.

### `AX_CODE_CONFIG_DIR`

Ruta al directorio de configuración de AX Code para agents, skills, snippets y descubrimiento de config.

### `AX_CODE_DATA_DIR`

Directorio de datos personalizado para el servidor AX Code gestionado.

### `AX_CODE_WSL_DISTRO`

Elige la distro WSL usada para la integración con AX Code en Windows.

### `OPENCHAMBER_AX_CODE_WSL_DISTRO`

Alias específico de AX Code App para elegir la distro WSL. `AX_CODE_WSL_DISTRO` tiene prioridad si ambas están definidas.

### `AX_CODE_JWT_SECRET`

Secreto para firmar tokens de autenticación de la UI. En servicios persistentes, usa un valor largo y aleatorio.

## Terminal y Git

### `OPENCHAMBER_TERMINAL_SHELL`

Ejecutable de shell para las sesiones de terminal en AX Code App.

### `OPENCHAMBER_GIT_BINARY`

Ejecutable de Git usado por las funciones Git de AX Code App.

### `GIT_BINARY`

Override alternativo para el ejecutable de Git. Para ajustes específicos de AX Code App, prefiere `OPENCHAMBER_GIT_BINARY`.

### `OPENCHAMBER_GIT_READ_CACHE_TTL_MS`

TTL, en milisegundos, para lecturas de archivos con cache respaldadas por Git. Usa `0` para desactivar la cache al depurar.

## Voz y túneles

### `OPENAI_API_KEY`

API key para funciones de voz de AX Code App que llaman a servicios compatibles con OpenAI.

### `OPENCHAMBER_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Permite base URLs remotas compatibles con OpenAI para funciones de voz cuando vale `true` o `1`.

### `NGROK_AUTHTOKEN`

Token de autenticación de ngrok usado por los comandos de túnel de AX Code App. También puedes configurarlo con `ngrok config add-authtoken <token>`.

## Ayudas de runtime

### `BUN_BINARY`

Ejecutable de Bun que AX Code App debe usar al iniciar procesos daemon.

### `BUN_INSTALL`

Raíz de instalación de Bun. AX Code App la usa para encontrar `bin/bun` al iniciar daemons y actualizar.

### `VITE_AX_CODE_URL`

API base URL de build-time para la web app construida con Vite. La mayoría de usuarios no debería usarla en CLI o desktop normales.
