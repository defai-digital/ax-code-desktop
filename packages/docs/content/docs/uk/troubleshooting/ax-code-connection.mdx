# Підключення до AX Code

Якщо AX Code App завантажується, але ніяк не виходить за межі стану «AX Code перезапускається», або чат не відповідає, сервер, із яким він спілкується, недоступний. Пройдіться по цьому.

## Він застряг на «AX Code перезапускається»

- дайте йому хвилину одразу після запуску — цей стан нормальний, поки сервер стартує
- перевірте, чи сервер живий, командою `openchamber status`
- перезапустіть його командою `openchamber restart`
- перегляньте деталі запуску командою `openchamber logs`

## Ви підключаєтеся до власного сервера

Якщо ви налаштували AX Code App на використання наявного сервера, ще раз перевірте налаштування з [AX Code Server](/uk/ax-code-server/):

- `AX_CODE_HOST` має містити порт, без шляху — наприклад, `http://localhost:4096`
- задайте `AX_CODE_SKIP_START=true`, щоб AX Code App не запускав ще й свій
- якщо адреса недійсна, AX Code App ігнорує її й натомість запускає власний сервер — шукайте попередження `[config]` у журналах

## Усе ще не працює

- переконайтеся, що версія Node.js — `20` або новіша
- перевстановіть найновіший CLI
- відкрийте `http://localhost:3000` напряму, перш ніж тестувати будь-який тунель чи віддалене посилання
