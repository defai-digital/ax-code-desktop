# Variáveis de ambiente

O AX Code App lê estas variáveis na inicialização. Para serviços de inicialização, `openchamber startup enable` salva por padrão um snapshot do ambiente atual, então execute de novo após mudar variáveis que o serviço deve usar.

## Servidor AX Code App

### `OPENCHAMBER_HOST`

Endereço onde o servidor web do AX Code App escuta. Use `0.0.0.0` para permitir acesso de outras máquinas.

### `OPENCHAMBER_UI_PASSWORD`

Senha da interface no navegador. Use quando expor fora do localhost, por túnel ou por proxy reverso.

### `OPENCHAMBER_DATA_DIR`

Altera o diretório de dados do AX Code App. O padrão é `~/.config/openchamber`.

### `OPENCHAMBER_COMPRESS_API`

Controla a compressão das respostas da API. `true` ou `1` ativa; `false` ou `0` desativa.

### `OPENCHAMBER_SKIP_API_COMPRESSION`

Desativa a compressão das respostas da API quando definido como `true` ou `1`. Tem prioridade sobre `OPENCHAMBER_COMPRESS_API`.

### `OPENCHAMBER_VERBOSE_REQUEST_LOGS`

Ativa logs HTTP detalhados quando definido como `true` ou `1`.

### `OPENCHAMBER_UPDATE_API_URL`

Altera o endpoint de verificação de atualizações. A maioria dos usuários deve deixar isso sem definir.

### `OPENCHAMBER_PACKAGE_MANAGER`

Força o gerenciador de pacotes usado em atualizações quando a detecção automática estiver errada.

## Servidor AX Code

### `AX_CODE_HOST`

Conecta o AX Code App a um servidor AX Code existente. O valor deve ser um origin `http` ou `https` com porta explícita e sem path, query ou hash. `AX_CODE_HOST` tem prioridade sobre `AX_CODE_PORT`.

### `AX_CODE_PORT`

Define a porta do servidor AX Code. Com AX Code gerenciado, solicita essa porta; com `AX_CODE_SKIP_START=true`, conecta a um servidor externo nessa porta.

### `AX_CODE_SKIP_START`

Impede o AX Code App de iniciar seu próprio servidor AX Code quando definido como `true`.

### `OPENCHAMBER_AX_CODE_HOSTNAME`

Hostname do servidor AX Code gerenciado pelo AX Code App. O padrão é `127.0.0.1`.

### `AX_CODE_BINARY`

Caminho para o executável `ax-code` que o AX Code App deve usar.

### `AX_CODE_CONFIG`

Caminho para um arquivo de configuração específico do AX Code.

### `AX_CODE_CONFIG_DIR`

Caminho para o diretório de configuração do AX Code para agents, skills, snippets e descoberta de config.

### `AX_CODE_DATA_DIR`

Diretório de dados personalizado para o servidor AX Code gerenciado.

### `AX_CODE_WSL_DISTRO`

Escolhe a distro WSL usada pela integração do AX Code no Windows.

### `OPENCHAMBER_AX_CODE_WSL_DISTRO`

Alias específico do AX Code App para escolher a distro WSL. `AX_CODE_WSL_DISTRO` tem prioridade quando ambas estão definidas.

### `AX_CODE_JWT_SECRET`

Segredo usado para assinar tokens de autenticação da UI. Em serviços persistentes, use um valor longo e aleatório.

## Terminal e Git

### `OPENCHAMBER_TERMINAL_SHELL`

Executável de shell usado nas sessões de terminal do AX Code App.

### `OPENCHAMBER_GIT_BINARY`

Executável do Git usado pelos recursos Git do AX Code App.

### `GIT_BINARY`

Override alternativo para o executável do Git. Para configuração específica do AX Code App, prefira `OPENCHAMBER_GIT_BINARY`.

### `OPENCHAMBER_GIT_READ_CACHE_TTL_MS`

TTL, em milissegundos, para leituras de arquivos em cache baseadas em Git. Use `0` para desativar o cache durante depuração.

## Voz e túneis

### `OPENAI_API_KEY`

API key usada por recursos de voz do AX Code App que chamam serviços compatíveis com OpenAI.

### `OPENCHAMBER_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Permite base URLs remotas compatíveis com OpenAI para recursos de voz quando definido como `true` ou `1`.

### `NGROK_AUTHTOKEN`

Token de autenticação do ngrok usado pelos comandos de túnel do AX Code App. Você também pode configurar com `ngrok config add-authtoken <token>`.

## Auxiliares de runtime

### `BUN_BINARY`

Executável do Bun que o AX Code App deve usar ao iniciar processos daemon.

### `BUN_INSTALL`

Raiz da instalação do Bun. O AX Code App usa isso para encontrar `bin/bun` ao iniciar daemons e atualizar.

### `VITE_AX_CODE_URL`

API base URL de build-time para a web app criada com Vite. A maioria dos usuários não deve usar isso no CLI ou desktop normal.
