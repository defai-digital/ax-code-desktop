# Połączenie z AX Code

Jeśli AX Code App się ładuje, ale nigdy nie przechodzi poza „AX Code uruchamia się ponownie”, albo czat nie odpowiada, serwer, z którym się komunikuje, jest nieosiągalny. Przejdź przez te kroki.

## Utknął na „AX Code uruchamia się ponownie”

- daj mu chwilę tuż po uruchomieniu — ten stan jest normalny, gdy serwer startuje
- sprawdź, czy serwer żyje, za pomocą `openchamber status`
- zrestartuj go za pomocą `openchamber restart`
- zobacz szczegóły startu za pomocą `openchamber logs`

## Łączysz się z własnym serwerem

Jeśli ustawiłeś AX Code App tak, aby używał istniejącego serwera, sprawdź ponownie konfigurację z [AX Code Server](/pl/ax-code-server/):

- `AX_CODE_HOST` musi zawierać port, bez ścieżki — na przykład `http://localhost:4096`
- ustaw `AX_CODE_SKIP_START=true`, aby AX Code App nie uruchamiał też własnego
- jeśli adres jest nieprawidłowy, AX Code App go ignoruje i uruchamia własny serwer — poszukaj ostrzeżenia `[config]` w logach

## Nadal nie działa

- potwierdź, że Node.js jest w wersji `20` lub nowszej
- przeinstaluj najnowsze CLI
- otwórz `http://localhost:3000` bezpośrednio, zanim przetestujesz jakikolwiek tunel lub zdalny link
