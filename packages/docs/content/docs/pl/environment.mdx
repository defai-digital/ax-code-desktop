# Zmienne środowiskowe

AX Code App odczytuje te zmienne przy starcie. Dla usług startowych `openchamber startup enable` domyślnie zapisuje snapshot bieżącego środowiska, więc uruchom tę komendę ponownie po zmianie zmiennych, których ma używać usługa.

## Serwer AX Code App

### `OPENCHAMBER_HOST`

Adres, na którym nasłuchuje serwer web AX Code App. Użyj `0.0.0.0`, aby pozwolić na dostęp z innych maszyn.

### `OPENCHAMBER_UI_PASSWORD`

Hasło do interfejsu w przeglądarce. Ustaw je przy dostępie spoza localhost, tunelach albo reverse proxy.

### `OPENCHAMBER_DATA_DIR`

Nadpisuje katalog danych AX Code App. Domyślnie jest to `~/.config/openchamber`.

### `OPENCHAMBER_COMPRESS_API`

Steruje kompresją odpowiedzi API. `true` albo `1` ją włącza, `false` albo `0` wyłącza.

### `OPENCHAMBER_SKIP_API_COMPRESSION`

Wyłącza kompresję odpowiedzi API, gdy ma wartość `true` albo `1`. Ma pierwszeństwo przed `OPENCHAMBER_COMPRESS_API`.

### `OPENCHAMBER_VERBOSE_REQUEST_LOGS`

Włącza szczegółowe HTTP request logs, gdy ma wartość `true` albo `1`.

### `OPENCHAMBER_UPDATE_API_URL`

Nadpisuje endpoint sprawdzania aktualizacji. Większość użytkowników nie powinna tego ustawiać.

### `OPENCHAMBER_PACKAGE_MANAGER`

Wymusza package manager używany do aktualizacji, gdy automatyczne wykrywanie się pomyli.

## Serwer AX Code

### `AX_CODE_HOST`

Łączy AX Code App z istniejącym AX Code server. Wartość musi być origin `http` albo `https` z jawnym portem i bez path, query ani hash. `AX_CODE_HOST` ma pierwszeństwo przed `AX_CODE_PORT`.

### `AX_CODE_PORT`

Ustawia port AX Code server. Dla managed AX Code prosi o ten port; z `AX_CODE_SKIP_START=true` łączy się z zewnętrznym serwerem na tym porcie.

### `AX_CODE_SKIP_START`

Gdy ma wartość `true`, blokuje uruchamianie własnego AX Code server przez AX Code App.

### `OPENCHAMBER_AX_CODE_HOSTNAME`

Hostname dla AX Code server zarządzanego przez AX Code App. Domyślnie `127.0.0.1`.

### `AX_CODE_BINARY`

Ścieżka do executable `ax-code`, którego ma używać AX Code App.

### `AX_CODE_CONFIG`

Ścieżka do konkretnego pliku config AX Code.

### `AX_CODE_CONFIG_DIR`

Ścieżka do katalogu config AX Code dla agents, skills, snippets i config discovery.

### `AX_CODE_DATA_DIR`

Własny katalog danych dla managed AX Code server.

### `AX_CODE_WSL_DISTRO`

Wybiera WSL distro używane do integracji AX Code w Windows.

### `OPENCHAMBER_AX_CODE_WSL_DISTRO`

Alias specyficzny dla AX Code App do wyboru WSL distro. `AX_CODE_WSL_DISTRO` ma pierwszeństwo, gdy ustawione są obie zmienne.

### `AX_CODE_JWT_SECRET`

Secret używany do podpisywania UI auth tokens. Dla trwałych usług użyj długiej losowej wartości.

## Terminal i Git

### `OPENCHAMBER_TERMINAL_SHELL`

Shell executable używany przez terminal sessions w AX Code App.

### `OPENCHAMBER_GIT_BINARY`

Git executable używany przez funkcje Git w AX Code App.

### `GIT_BINARY`

Alternatywny override dla Git executable. Dla konfiguracji specyficznej dla AX Code App preferuj `OPENCHAMBER_GIT_BINARY`.

### `OPENCHAMBER_GIT_READ_CACHE_TTL_MS`

TTL w milisekundach dla cached Git-backed file reads. Ustaw `0`, aby wyłączyć cache podczas debugowania.

## Głos i tunele

### `OPENAI_API_KEY`

API key używany przez funkcje głosowe AX Code App, które wywołują OpenAI-compatible services.

### `OPENCHAMBER_ALLOW_REMOTE_OPENAI_COMPAT_URLS`

Pozwala na zdalne OpenAI-compatible base URLs dla funkcji głosowych, gdy ma wartość `true` albo `1`.

### `NGROK_AUTHTOKEN`

ngrok auth token używany przez tunnel commands AX Code App. Możesz też skonfigurować go przez `ngrok config add-authtoken <token>`.

## Runtime helpers

### `BUN_BINARY`

Bun executable, którego AX Code App ma używać do uruchamiania daemon processes.

### `BUN_INSTALL`

Katalog główny instalacji Bun. AX Code App używa go, aby znaleźć `bin/bun` dla daemon startup i aktualizacji.

### `VITE_AX_CODE_URL`

Build-time API base URL dla Vite-built web app. Przy normalnym użyciu CLI lub desktop większość użytkowników nie powinna tego ustawiać.
