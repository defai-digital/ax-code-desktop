# Remote Access

When you can't reach AX Code Desktop from another trusted workstation, the fix depends on how you're connecting.

## Check the basics first

- open `http://localhost:3000` on the same computer first — if that fails, it's not a remote problem; see [AX Code connection](/troubleshooting/ax-code-connection/)
- confirm the server is running with `ax-code-desktop status`

## Remote instance won't connect (desktop)

When a [remote instance](/remote-instances/) stalls, AX Code Desktop names the step that failed:

- **auth** — your SSH or UI password was rejected; re-enter it
- **install / start** — AX Code Desktop couldn't set up or start the server on the remote machine; check that machine's requirements
- **forwarding** — the connection is up but the port isn't reaching you; try a different local port

## Behind your own server

If you put AX Code Desktop behind a reverse proxy and it loads oddly or won't connect, see [Reverse Proxy](/reverse-proxy/).
