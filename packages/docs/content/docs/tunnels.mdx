# Tunnels

A tunnel is a public link to your AX Code Desktop for controlled remote workstation access. Use `ax-code-desktop tunnel` to create one for a running instance.

## Prerequisites

AX Code Desktop starts the tunnel provider CLI on your machine. Install the provider you want to use first:

```bash
brew install cloudflared
brew install ngrok
```

Cloudflare quick tunnels can run with `cloudflared`. Ngrok requires an ngrok account and an authtoken from the ngrok dashboard:

```bash
ngrok config add-authtoken <your-ngrok-token>
```

## Quick start

1. Start AX Code Desktop:

```bash
ax-code-desktop
```

2. Start a Cloudflare tunnel:

```bash
ax-code-desktop tunnel start --provider cloudflare --mode quick
```

Or start an Ngrok tunnel:

```bash
ax-code-desktop tunnel start --provider ngrok --mode quick
```

3. Check status:

```bash
ax-code-desktop tunnel status
```

When the tunnel is up, `status` shows a public URL. Open it from a trusted workstation.

Copy the generated connect link only to trusted workstation browsers.

## Providers

- `cloudflare`: quick, managed remote, and managed local modes
- `ngrok`: quick mode

## Managed modes

### Managed remote

Use a token + hostname managed by Cloudflare:

```bash
ax-code-desktop tunnel start --provider cloudflare --mode managed-remote --token-file ~/.secrets/cf-token --hostname app.example.com
```

### Managed local

Use a local `cloudflared` config:

```bash
ax-code-desktop tunnel start --provider cloudflare --mode managed-local --config ~/.cloudflared/config.yml
```

## Profiles (managed-remote)

Save a reusable profile:

```bash
ax-code-desktop tunnel profile add --provider cloudflare --mode managed-remote --name prod-main --hostname app.example.com --token-file ~/.secrets/cf-token
```

Start using the saved profile:

```bash
ax-code-desktop tunnel start --profile prod-main
```

## Useful commands

```bash
ax-code-desktop tunnel providers
ax-code-desktop tunnel ready --provider cloudflare
ax-code-desktop tunnel ready --provider ngrok
ax-code-desktop tunnel doctor --provider cloudflare
ax-code-desktop tunnel doctor --provider ngrok
ax-code-desktop tunnel stop --port 3000
```

## Behavior notes

- one active tunnel per AX Code Desktop instance (port)
- starting a new mode/provider on same instance replaces previous tunnel
- generating a new connect link revokes previous unused one
